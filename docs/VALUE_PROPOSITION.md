# NutriSnap — Value Proposition

## Главный месседж

> **Все приложения для дневника питания превращают пользователя в бухгалтера. NutriSnap делает AI-диетолога.**

FatSecret / Calz / MyFitnessPal **считают** калории. NutriSnap **помогает** их съесть правильно.

---

## Где конкуренты проигрывают

| Что | FatSecret / Calz | Где провал |
|---|---|---|
| Логирование | работает, но медленно — ручной поиск + ввод граммов | мы решаем: AI ввод (фото/голос/текст/форвард) |
| Напоминания | тупой scheduler: "18:00 — поешь" | не учитывают: поел ли уже, в типичное ли время, какой остаток ккал |
| Цели | "съешь 2000 ккал" без контекста | не объясняют **как именно** и **что конкретно** |
| Рекомендации | **нет** | главный пробел — никто не подсказывает что съесть |
| Анализ | графики калорий, скучные сводки | без выводов и actionable советов |
| Однообразие рациона | игнорируется | юзер ест одно и то же 2 недели → бросает приложение |

Результат: люди ставят app → используют 1-2 недели → бросают. Потому что приложение не помогает достичь цели, только считает прогресс.

---

## Решение: 3 AI-нативные фичи

### F13 — Smart pattern-based reminders

❌ FatSecret: "18:00 — время ужина" (даже если ты в кино / поел в 17 / не голоден)

✅ NutriSnap: "Ты обычно ужинаешь около 19:30, сейчас 20:45. У тебя осталось 800 ккал — возьми что-то лёгкое 🥗"

**Под капотом:**
- Анализ истории `eaten_at` → профиль типичного времени per meal_type (avg + std)
- PTB JobQueue раз в час → проверка: было ли логирование в типичное окно?
- LLM генерирует уникальный текст с контекстом дня (не шаблон)
- Если уже залогил — молчит
- Если пропустил всё утро — не давит, спрашивает мягко

**A/B эксперимент:** retention при тупом scheduler vs AI напоминаниях.

### F14 — AI Nutritionist (`/recommend`)

```
Юзер: /recommend
Bot: 📊 За сегодня: 1400/2200 ккал, белок 60/130г, жир 50/70г, углеводы 180/250г

     🎯 Тебе не хватает: ~800 ккал, +70г белка

     Варианты ужина:
     1️⃣ Курица 200г + гречка 100г + овощи — 470 ккал, 53г белка
     2️⃣ Лосось 150г + рис 80г — 510 ккал, 38г белка  ← новое
     3️⃣ Творог 5% 200г + орехи 30г + банан — 480 ккал, 35г белка  ← новое

     Что выбираешь? (или напиши своё)
```

**Под капотом:**
- LLM получает: target КБЖУ юзера, сводка за день, что ел недавно, кураторская база `foods`
- RAG из `foods` подтягивает быстрые/доступные блюда под цель
- Возвращает 3 варианта: один из истории юзера (комфорт), два новых (вариативность)

**Главный USP проекта.** Это закрывает RAG + LangGraph workflow + полную выгоду от мультимодальности.

### F15 — Variety nudges

```
👀 Заметил: ты 5 дней подряд ел овсянку на завтрак.

Может попробуешь?
[🍳 Яичница] [🥑 Тост с авокадо] [🥤 Смузи]

Или [Свой завтрак как обычно] →
```

**Под капотом:**
- SQL: TOP-1 product per meal_type за 7 дней, если повторов >= 5
- LLM подбирает 3 альтернативы с похожими КБЖУ
- Тон: "может попробуешь?", не "ты неправильно ешь"

**Зачем:** разнообразие = retention. Calz/FatSecret вообще про это не думают.

## Как продавать менторам на защите

1. Открыть с вопроса: **"Почему 90% юзеров бросают MyFitnessPal через 2 недели?"**
2. Ответ: **"Потому что приложение их не учит — оно их учитывает."**
3. Демо: показать как NutriSnap делает три вещи которых нет ни у кого:
   - Умное напоминание с контекстом
   - `/recommend` с тремя вариантами под остаток дня
   - Variety nudge "ты ел овсянку 5 дней — попробуй?"
4. Все три фичи **физически невозможны без LLM-контекста** → закрывает критерий "Качество LLM-инженерии" (25%).
5. Бизнес-ценность: **AI-диетолог в Telegram стоит ~5000₸/мес от живого диетолога. Мы это даём за $0** (или $3/мес с подпиской на премиум).
